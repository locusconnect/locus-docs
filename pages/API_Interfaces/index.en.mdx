# API Interfaces

The Locus RTLS system provides several powerful API interfaces, enabling developers to seamlessly integrate real-time location data and system controls into custom applications and existing enterprise systems. This documentation covers the available APIs, authentication methods, request/response formats, and common usage scenarios to help you get started quickly.

## Available APIs

| API | Purpose | Protocol | Documentation |
|-----|---------|----------|---------------|
| Positioning API | Real-time position data access | WebSocket/REST | [Details](#positioning-api) |
| Configuration API | System configuration | REST | [Details](#configuration-api) |
| Tag Management API | Tag registration and control | REST | [Details](#tag-management-api) |
| Historical Data API | Access to historical position data | REST | [Details](#historical-data-api) |
| Status API | System health and diagnostics | REST | [Details](#status-api) |

## Authentication

All Locus APIs use token-based authentication:

```bash
curl -X POST https://[your-locus-server]/api/auth/login 
   -H "Content-Type: application/json" 
   -d '{"username": "your_username", "password": "your_password"}'
```

The response will include a JWT token for subsequent requests:

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires": "2023-12-31T23:59:59Z"
}
```

Include this token in the Authorization header for all API requests:

```bash
curl -X GET https://[your-locus-server]/api/positions
   -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

## Positioning API

The Positioning API provides real-time access to tag position data. It supports both WebSocket connections for live data streaming and REST endpoints for polling.

### WebSocket Example

```javascript
const socket = new WebSocket('wss://[your-locus-server]/api/positions/live');

socket.onopen = function(e) {
  socket.send(JSON.stringify({
    "token": "your_authentication_token",
    "tags": ["tag1", "tag2"] // Optional filter
  }));
};

socket.onmessage = function(event) {
  const positionData = JSON.parse(event.data);
  console.log(positionData);
};
```

### Response Format

```json
{
  "tagId": "asset_123",
  "timestamp": "2023-09-17T14:23:45.123Z",
  "position": {
    "x": 12.34,
    "y": 45.67,
    "z": 1.23
  },
  "accuracy": 0.15,
  "additionalData": {
    "battery": 85,
    "temperature": 23.5,
    "motion": true
  }
}
```

## Configuration API

The Configuration API allows you to programmatically manage and configure settings for the Locus RTLS system, including parameters for anchors, tags, and overall system behavior.

## Tag Management API

The Tag Management API provides endpoints for registering new tags, updating existing tag information, and managing the lifecycle of tags within your RTLS deployment.

## Historical Data API

The Historical Data API allows you to query, retrieve, and export historical position data for in-depth analysis, reporting, and auditing purposes.

## Status API

The Status API provides access to real-time system health information, operational status of components, and diagnostic data for effective monitoring and maintenance.
